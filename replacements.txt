# ── NOTE ──────────────────────────────────────────────────────────────────────
# The following four variables are consumed via $env['...'] in Code nodes and
# do NOT appear as [BRACKET] tokens in the JSON. They are mandatory.
# Omitting any of these will cause runtime errors in the Code nodes that
# reference them.
# ──────────────────────────────────────────────────────────────────────────────

BAMBOOHR_HMAC_SECRET
  Value:  HMAC-SHA256 secret from BambooHR Settings → Webhooks
  Where:  $env['BAMBOOHR_HMAC_SECRET'] in 01-hris-intake-bamboohr.json → Code node
          "Verify HMAC + Normalize Payload"
  How to get: BambooHR → Settings → Webhooks → create or view webhook → copy
              the Secret field (shown only once at creation time; regenerate if lost)
  Note:   Store ONLY in n8n Settings → Variables. If you hardcode this in a
          node field it will appear in execution logs in plain text.

GSUITE_SERVICE_ACCOUNT_JSON
  Value:  Full JSON content of the Google service account key file
  Where:  $env['GSUITE_SERVICE_ACCOUNT_JSON'] in 04-account-provisioning.json →
          Code node "Provision All Accounts" (GSuite section)
  How to get: Google Cloud Console → IAM & Admin → Service Accounts →
              select your service account → Keys → Add Key → JSON →
              copy the entire file content as a single-line JSON string
  Note:   The service account must have domain-wide delegation enabled and the
          scope https://www.googleapis.com/auth/admin.directory.user granted in
          Google Admin → Security → API Controls → Domain-wide delegation.

GSUITE_ADMIN_EMAIL
  Value:  Email address of a Google Workspace super-admin
          (e.g., admin@yourcompany.com)
  Where:  $env['GSUITE_ADMIN_EMAIL'] in 04-account-provisioning.json →
          Code node "Provision All Accounts" (GSuite JWT assertion)
  How to get: Google Admin Console → Account → Admin Roles → Super Admin →
              note the primary email of the admin account used for impersonation
  Note:   The service account JWT will impersonate this address. The account
          must remain active; deactivating it breaks GSuite provisioning.


SLACK_INVITE_MODE  [env]
  Value:  "legacy" (default) or "scim"
  Where:  $env reference in 04-account-provisioning "Provision All Services" Code
          node (Slack branch) — controls which Slack API endpoint is used for
          user provisioning
  How to get: Determine per client:
              - "legacy" for Enterprise Grid or Legacy Admin workspaces
                (users.admin.invite is available)
              - "scim" for standard paid workspaces (Business+, Pro) with
                SCIM provisioning enabled
              If the variable is unset, the Code node defaults to "legacy".
  Note:   SCIM mode requires additional body structure changes to the Slack
          branch fetch call AND a separate SLACK_SCIM_TOKEN variable.
          See SETUP.md §11 for full SCIM configuration.
